def plotPerColumnDistribution(df, nGraphShown, nGraphPerRow , cols_index):
    nRow, nCol = df.shape
    subnum = 0
    columnNames = list(df)
    nGraphRow = (nCol + nGraphPerRow - 1) / nGraphPerRow
    plt.figure(num = None, figsize = (6 * nGraphPerRow, 8 * nGraphRow), dpi = 80, facecolor = 'w', edgecolor = 'k')
    for i in cols_index:
        subnum += 1
        plt.subplot(nGraphRow, nGraphPerRow, subnum )
        columnDf = df.iloc[:, i]
        if (not np.issubdtype(type(columnDf.iloc[0]), np.number)):
            valueCounts = columnDf.value_counts()
            valueCounts.plot.bar()
        else:
            columnDf.hist()
        plt.ylabel('counts')
        plt.title(f'{columnNames[i]} (column {i})')
    plt.tight_layout(pad = 1.0, w_pad = 1.0, h_pad = 1.0)
    plt.show();

Hi there,

After having a deep look into the 3 datasets you have provided, I have made some conclusions about the data quality and I got some advice for how to improve your datasets for better results when using it for segmentation and decision making.

Here are some of the Quality issues I found in the datasets :

  • Missing address values for some customers.
  • Missing values in other columns.
  • Missing values registed with different names like the string 'nan'.
  • Erroness values (like the date of birth of one customer being 1843-12-21).
  • Inconsistent values in the Default columns.
  • Inaccurate different values for the same gender in the gender column ( Female = Female, Femal, F and Male = Male , M)
  • Different values for Binary true and false values. (Y/N , Yes/No ). Should use True/False
  • Column country contain one unique value 'Australia'. Irrelevant data should be dropped.
  • Inaccurate different values for States (NSW/New South Whales , VIC/Victoria).

all these Quality issues have been fixed above.

My advice for you is to pay attention to these mistakes as it can lead to false interpretations of the data.
